## Key Findings

### Performance Summary
- Fewer than 350 letter-to-sound rules produce correct IPA transcriptions for the great majority of English words
- Of the 1000 most frequent English words, the process mispronounces fewer than 4%
- Words are counted according to their frequency of occurrence
- Error rate rises with decreasing word frequency
- Since over 2/3 of words in a typical sample are among the most frequent thousand, relatively poor performance on rare words does not drive overall error rate higher than about 10%
- On average, the program mispronounces fewer than two words per sentence of ordinary written English
- Most mispronunciations are single-phoneme errors and easily correctable from context

### Version Evolution
- Version 1 to Version 3: error rate for 1000 most frequent words reduced from 32% to 4%
- Number of rules increased by three quarters during this evolution

### Comparison with Ainsworth
- Version 1 frequency-weighted score: 68% (from Table 5)
- Ainsworth's reported scores: 89% to 92%
- Discrepancy attributed to:
  1. British vs American pronunciation differences (r-dropping, etc.)
  2. Subjectivity in judging acceptable pronunciations
  3. Sensitivity to vocabulary level and text style

### Inflection and Prosody
- Current system produces flat monotone that is fatiguing
- Simple heuristics for inflection tested: stress, pitch, rhythm, and timing all important
- Alternating stress significantly preferred over random stress or monotone
- Hand-inserted "correct" English stress significantly preferred over all others
- Stress assignment scheme based on:
  1. Falling inflection before period, rising before question mark
  2. Rules with schwa output are good predictors for stressing/destressing
  3. Common function words and common endings (like ES, ED) predict unstressed syllables
  4. English speech tends to stress approximately alternating syllables
- Unstressed syllables: lower pitch, shorter duration
- Stressed syllables: timing adjusted by further reduction of adjacent unstressed syllables and lengthening adjacent stressed syllables

### Navy Application
- Working on tailoring rules for Navy-specific vocabulary
- Acronyms like WWMCCS (/wImIks/) require special handling
- Simple expedient: pronounce each consonant as its name when context is entirely consonants
- Already catches acronyms like NRL

## Rule Format (Detailed from Program Listing)

### Basic Form
```
A[B]C=D
```
Where:
- A = left context (string of alphabetics or special symbols)
- B = letters to be matched (must be alphabetic only)
- C = right context (string of alphabetics or special symbols)
- D = phoneme output

### Special Symbol Definitions (from pages 50, 73, 76)
| Symbol | Meaning |
|--------|---------|
| `#` | 1 or more vowels |
| `*` | 1 or more consonants |
| `.` | A voiced consonant |
| `$` | Single consonant followed by 'I' or 'E' |
| `%` | Suffix such as 'E', 'ES', 'ED', 'ER', 'ING', 'ELY' |
| `&` | A sibilant |
| `@` | A consonant after which long 'U' is pronounced (as in 'RULE', not 'MULE') |
| `^` | A single consonant |
| `+` | A front vowel: 'E', 'I', 'Y' |
| `:` | 0 or more consonants |

## Algorithms/Procedures

### TRANSLATETEXT Algorithm (page 72)
1. Start scan at first character of input (position 0 is blank inserted by program to delimit first word)
2. Initialize I = 1, TRANSLATETEXT = NULL
3. Loop NEXTCHAR:
   - Pick off one character of 'TEXT' at Ith position into CHAR
   - Test for end text marker - if so return
   - Concatenate the phoneme returned by TRANSLATE: `TRANSLATETEXT = TRANSLATETEXT TRANSLATE(TEXT,CHAR,'ENG')`
   - Increment pointer to next character: `I = I + INCVALUE`
   - INCVALUE is set by routine TRANSLATE

### TRANSLATE Algorithm (pages 73-75)
1. Check if GRAPHEME is punctuation symbol or digit - set QUAL to 'PUNCT' or 'NUMBER'
2. Build rule string name from GRAPHEME and QUAL (e.g., ARULE.ENG for 'A' with 'ENG')
3. Copy set of possible rules: `GRRULE = $(GRAPHEME 'RULE.' QUAL)`
4. Loop NEXTRULE - break off rules from rule string (delimited by '\')
5. For each rule:
   - Break into BACKCHAR, CHARDEF, FORCHAR, PHONEME
   - If special symbols (#, *, etc.) in BACKCHAR or FORCHAR, call SPECIALCASEPROC
   - Otherwise match directly on the pieces
   - Match by backing up SIZE(BACKCHAR) characters in buffer
   - If match succeeds, return PHONEME and set INCVALUE = SIZE(CHARDEF)
6. If no rule matches: output error message

### SPECIALBREAK Algorithm (page 76)
1. Define character classes for special symbols
2. Loop REMATCH: scan string for special case characters
3. For each special character found:
   - Break off into SYM
   - Build pattern from $('PATTERN' SYM)
   - Concatenate: `PATTERN = PATTERN PATTERN1 $('PATTERN' SYM)`
4. Return complete pattern with remaining string appended

1. The rules (ARULE.ENG, BRULE.ENG, ..., ZRULE.ENG, NUMBERRULE.ENG, PUNCTRULE.ENG)
2. Function-name declarations and initialization
3. Translation routines (TRANSLATETEXT, VOTRAXTRANSLATE, TRANSLATE)
4. Service routines (FILEDEFINE, CLI, FILEOUT, ASCII)

### Rule Organization
- Rules stored in variables named `XRULE.ENG` for English-to-IPA
- Rules stored in variables named `XRULE.IPA` for IPA-to-Votrax
- Rules delimited by `\` (backslash) within each variable
- Each rule has format: `context[letter]context=phoneme`

### Key Implementation Details
1. A blank is inserted before input string to delimit first word
2. End-of-text marker is `#`
3. Escape code for file termination is `###`
4. Rules are tried in order - first match wins
5. Special symbols in rules are expanded to SNOBOL patterns at runtime
6. Translation can go English->IPA->Votrax->ASCII in sequence

### Platform
- Written in SNOBOL for PDP10
- Output compatible with TI 960A speech lab
- Can output to TI 733 cassette format
